# 设计文档：星枢令 Apple 风格 GitHub 发布页

- **日期**：2026-06-08
- **状态**：已批准（待 spec 复核）
- **产物**：`docs/release/index.html`（单文件，自包含）

## 目标

为星枢令（ArcaneKey，跨平台 TOTP 身份验证器）构建一个采用苹果产品官网风格的发布/落地页，托管于现有 GitHub Pages（`mengyuefeitian.github.io/ArcaneKey/release/`）。

**成功标准：**
- 在浏览器中打开 `docs/release/index.html` 即可完整展示，无需构建、无外部网络依赖。
- 视觉风格明显呈现苹果官网气质：大留白、超大粗体标题、明暗章节交替、产品大图、滚动淡入动效。
- 完整覆盖产品卖点（动态验证码、扫码、主题、本地优先隐私、双平台），并提供三个获取入口。
- 移动端与桌面端均正常布局。

## 非目标（YAGNI）

- 不做多语言切换（仅中文）。
- 不做真实应用截图（用 CSS 还原 UI，截图位后续可替换但本期不依赖）。
- 不接入分析/埋点、不引入任何 JS 库或网络字体。
- 不改动现有 `privacy.html` 或仓库 README。
- 不替换 GitHub Pages 站点首页（落地页位于 `release/` 子目录）。

## 决策记录

## 视觉语言

- **字体栈**：`-apple-system, BlinkMacSystemFont, "SF Pro Display", "PingFang SC", "Helvetica Neue", system-ui, sans-serif`。无网络字体。
- **强调色**：海洋蓝 `#4080D0`（产品 `THEMES[0]` 的十六进制等价）。
- **暗色背景**：`#0d0d12` / `#191920`（与产品 App 一致）。
- **明色背景**：`#fbfbfd`（苹果官网常用近白）+ 文本 `#1d1d1f`。
- **章节节奏**：明 → 暗 → 明交替，制造苹果式呼吸感。
- **动效**：
  - 滚动进入视口淡入上浮（IntersectionObserver，`opacity` + `translateY`）。
  - 锚点平滑滚动（`scroll-behavior: smooth`）。
  - 倒计时环 SVG/CSS 呼吸动画（纯装饰）。
  - 所有动效遵守 `prefers-reduced-motion`，关闭时不位移。

## 核心组件：CSS 手机模型（`.device`）

纯 CSS/HTML 渲染，无图片：
- 外壳：圆角 ~44px、深色边框、灵动岛状态条、底部投影 + 主题色光晕。
- 屏幕内容（简化还原星枢令主屏）：
  - 深色背景 `#0d0d12`。
  - 顶部标题「星枢令」。
  - 2–3 张验证码卡片：品牌字母头像（圆形，取 `BRAND_COLORS` 风格的纯色）+ 账号名 + 6 位验证码以 `123 456` 分组 + 右侧倒计时环。
  - 底部玻璃态悬浮胶囊导航（首页/扫一扫/我的，`backdrop-filter: blur`）。
- 做成可复用结构；通过少量变体类切换展示内容（如主题色变体用于"10 款主题"章节）。

## 页面结构

1. **顶部导航（sticky）**：毛玻璃半透明吸顶（`backdrop-filter: blur`），左侧 logo + 「星枢令」，右侧锚点链接（功能 / 平台 / 隐私 / 获取）+ 小号「获取」按钮。小屏折叠为精简版。
2. **Hero**：巨幅标题「星枢令」+ 副标题「你的密钥，只属于这台设备。」+ 一行说明 + 主 CTA「在 GitHub 获取」(真实链接) + 次级链接「了解更多」(锚点)。下方悬浮主屏手机模型，带主题色光晕。
3. **特性聚焦**：苹果式「大字 + 视觉」交替行（图文左右交错）：
   - 动态验证码 · 倒计时环（手机模型聚焦卡片 + 倒计时环动画）。
   - 扫码即添加（扫码取景框视觉）。
   - 10 款主题（横排色板 swatches，点击/hover 切换手机模型主题色）。
   - 本地优先 · 隐私安全（密钥不上传服务器的图形化表达）。
4. **特性网格（Bento）**：小卡片网格，复用现有 SVG 图标（手动输入、加密备份、快速搜索、手势操作）。SVG 以内联或相对路径引用 `../../image/*.svg`（需确认可访问，否则内联）。
5. **双平台**：两张卡片 — 鸿蒙原生（ArkTS、玻璃态悬浮导航、ScanKit）vs 微信小程序（免安装、扫码即添加、云函数反馈），各列亮点。
6. **获取**：三入口横排卡片：
   - GitHub：真实链接 `https://github.com/mengyuefeitian/ArcaneKey`。
   - 微信小程序码：二维码占位卡（带 `<!-- TODO 替换为真实小程序码 -->` 注释与占位图形）+ 文案「微信搜索：星枢令」。
   - 鸿蒙应用市场：按钮占位（带 TODO 注释，链接 `#`）。
7. **页脚**：隐私政策链接（→ `../privacy.html`）、MIT 许可、`© 2026 ArcaneKey`、GitHub 链接。

## 技术与文件

- 单文件 `docs/release/index.html`：
  - `<head>`：meta viewport、`<title>星枢令 · ArcaneKey</title>`、内联 `<style>`。
  - `<body>`：上述 7 个章节的语义化结构（`<header><section><footer>`）。
  - 底部内联 `<script>`：IntersectionObserver 滚动淡入、主题色板切换、锚点（CSS 已处理则 JS 最小化）。
- 无构建步骤、无 npm、无外部 CDN、无网络字体。
- 图标策略：优先内联 SVG（避免相对路径在 `release/` 子目录失效）；若引用现有 `image/*.svg`，使用相对路径 `../../image/<name>.svg` 并在实现时验证可达。

## 响应式

- 移动端优先；断点约 `768px`。
- 大屏：图文交替左右布局；小屏：纵向堆叠，手机模型缩放。
- 导航在小屏精简（隐藏次要锚点或缩为图标）。

## 可访问性

- 语义化标签、`alt`/`aria-label`、可聚焦的链接按钮。
- 颜色对比满足正文可读（暗色背景上浅色文本）。
- `prefers-reduced-motion` 时禁用位移动效。

## 验证方式

- 在浏览器打开 `docs/release/index.html`，逐章节目视核对：hero、特性聚焦、网格、双平台、获取、页脚均正确渲染。
- 缩放至移动端宽度（≤414px）确认堆叠正常、无横向溢出。
- 控制台无报错；GitHub 链接可点击跳转；占位项带明确 TODO 注释。
- 离线打开（断网）仍完整渲染，证明无外部依赖。
