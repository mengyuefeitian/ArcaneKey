鸿蒙 6.1（API 23）的**沉浸光感（Immersive Light Effects）**和**底部悬浮菜单（Floating Tabs）**是 ArkUI 框架在 `@kit.UIDesignKit` 中新增的重磅特性。

这两个功能主要通过 **`HdsTabs`** 组件配合 **`.barFloatingStyle`** 和 **`systemMaterialEffect`** 属性来实现。

以下是基于最新 API 23 的完整代码示例和实现逻辑：

### 1. 核心代码实现

你需要导入 `@kit.UIDesignKit`，并使用 `HdsTabs` 组件。

```typescript
import { hdsMaterial, HdsTabs, HdsTabsController } from '@kit.UIDesignKit';
import { SymbolGlyphModifier, SymbolRenderingStrategy } from '@kit.ArkUI';

@Entry
@Component
struct ImmersiveTabsDemo {
  @State currentIndex: number = 0;
  private hdsTabsController: HdsTabsController = new HdsTabsController();

  // 定义菜单项（使用系统符号图标）
  private menuItems: Array<{ symbol: SymbolGlyphModifier, label: string }> = [
    { 
      symbol: new SymbolGlyphModifier($r('sys.symbol.house_fill')).fontColor(Color.Blue), 
      label: '首页' 
    },
    { 
      symbol: new SymbolGlyphModifier($r('sys.symbol.search')).fontColor(Color.Gray), 
      label: '发现' 
    },
    { 
      symbol: new SymbolGlyphModifier($r('sys.symbol.person_fill')).fontColor(Color.Gray), 
      label: '我的' 
    }
  ];

  build() {
    Stack() {
      // 1. 页面主要内容区域
      // 为了演示悬浮效果，这里放一个长列表
      List({ space: 10 }) {
        ForEach(Array(20), (item, index) => {
          ListItem() {
            Text(`列表项 ${index + 1}`)
              .width('100%')
              .height(100)
              .backgroundColor('#F1F3F5')
              .textAlign(TextAlign.Center)
              .fontSize(18)
          }
        })
      }
      .width('100%')
      .padding({ bottom: 80 }) // 预留底部空间，防止内容被遮挡（虽然悬浮栏不占位，但为了视觉舒适）

      // 2. 底部悬浮页签栏
      HdsTabs({ controller: this.hdsTabsController }) {
        ForEach(this.menuItems, (item, index) => {
          TabContent() {
            // 这里的内容会被 HdsTabs 自动管理，通常留空，因为内容区由外部 Stack 控制
          }
          .tabBar(item.symbol) // 设置图标
          .barStyle({ // 设置未选中和选中状态的样式
            iconSize: 24,
            fontColor: index === this.currentIndex ? Color.Blue : Color.Gray,
            selectedFontColor: Color.Blue
          })
        })
        .vertical(false) // 水平排列
        .barPosition(BarPosition.End) // 底部位置
        .onChange((index: number) => {
          this.currentIndex = index;
        })
      }
      // --- 核心配置开始 ---
      .barOverlap(true) // 【关键】允许页签栏与内容重叠（悬浮）
      .barFloatingStyle({
        barWidth: { 
          smallWidth: 300,  // 小屏宽度
          mediumWidth: 400, // 中屏宽度
          largeWidth: 500   // 大屏宽度
        },
        barBottomMargin: 28, // 距离底部的边距
        // 【核心】沉浸光感材质配置
        systemMaterialEffect: {
          materialType: hdsMaterial.MaterialType.ADAPTIVE, // 自适应材质（推荐）
          materialLevel: hdsMaterial.MaterialLevel.ADAPTIVE // 自适应等级
        },
        // 可选：渐变遮罩，让底部内容过渡更自然
        gradientMask: {
          maskColor: '#66F1F3F5',
          maskHeight: 92
        }
      })
      // --- 核心配置结束 ---
      .position({ x: 0, y: 0 }) // 配合 Stack 定位
    }
    .width('100%')
    .height('100%')
  }
}
```

### 2. 关键属性详解

要实现你看到的“高级感”，请重点关注以下配置：


| 属性                       | 说明                                                                           | 推荐值         |
| :------------------------- | :----------------------------------------------------------------------------- | :------------- |
| **`.barOverlap(true)`**    | 必须设置为`true`，否则页签栏会占据布局空间，无法实现“悬浮”在内容之上的效果。 | `true`         |
| **`barBottomMargin`**      | 设置底部边距，让悬浮栏“浮”起来，不要贴底。                                   | `28` (vp)      |
| **`systemMaterialEffect`** | 开启沉浸光感的核心。                                                           | 见下方材质说明 |

### 3. 材质类型选择（光感强弱）

在 `systemMaterialEffect` 中，你可以选择不同的材质等级来控制“光感”的强弱：

* **推荐方案（自适应）：**

  * `materialType: hdsMaterial.MaterialType.ADAPTIVE`
  * `materialLevel: hdsMaterial.MaterialLevel.ADAPTIVE`
  * **优点**：系统会根据设备性能自动调整，保证流畅度。
* **进阶方案（自定义光感）：**

  * **`IMMERSIVE` (沉浸)**：效果最强，有细腻的光晕和反射，但性能开销大。
  * **`EXQUISITE` (精致)**：光影反馈强烈。
  * **`GENTLE` (柔和)**：光影反馈内敛。
  * **`SMOOTH` (平滑)**：性能开销最低，适合低端设备降级使用。

### 4. 避坑指南

1. **版本要求**：必须使用 **DevEco Studio 6.0+** 且 SDK 版本为 **API 23 (HarmonyOS 6.1)** 或以上。
2. **真机调试**：部分沉浸光感特效（特别是模糊和光照）在**模拟器**上可能显示为纯色块或无效果，**请务必使用真机运行**查看最终效果。
3. **类型断言**：如果在 ArkTS 4.0 严格模式下遇到深层嵌套对象的报错，请记得使用 `as HdsTabsFloatingStyle` 进行类型断言。

你可以直接将上述代码复制到 DevEco Studio 的 `Index.ets` 中运行，即可看到带有磨砂玻璃质感和光感的悬浮底部菜单。
