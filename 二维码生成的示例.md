是的，扫码时二维码里已经包含了所有参数。标准的 `otpauth://` URI 格式如下：

```
otpauth://totp/{issuer}:{account}?secret={secret}&issuer={issuer}&algorithm={algo}&digits={digits}&period={period}
```

Authenticator 解码后就能拿到全部字段。下面是一个完整的 Python 示例：代码运行成功，附上示例二维码（用示例参数生成的）。

**核心逻辑只有两步：**

1. **拼 URI** — 把所有参数按格式塞进 `otpauth://totp/...`，这是协议规定的格式，所有 Authenticator 都认
2. **渲染成图** — 用任意二维码库把这段 URI 编码成图片

**参数来源对照：**

| URI 字段 | 对应你从 app 获取的 |
|---|---|
| `label`（路径部分）| issuer + account |
| `secret` | secret 编码 |
| `algorithm` | 通常是 SHA1，app 里一般不特别显示 |
| `digits` | 通常是 6 位 |
| `period` | 通常是 30 秒 |

如果 app 扫码时没有额外显示 algorithm/digits/period，直接用默认值（SHA1/6/30）就对，99% 的网站都是这套。

```Python
"""
TOTP 二维码重建示例
依赖：pip install qrcode[pil]

原理：
  Authenticator 扫码时读取的是一段 otpauth:// URI
  只要构造出相同的 URI，生成的二维码就与原始完全等效
"""

import urllib.parse
import qrcode


def build_otpauth_uri(
    account: str,
    secret: str,
    issuer: str = "",
    algorithm: str = "SHA1",
    digits: int = 6,
    period: int = 30,
) -> str:
    """
    构造标准 otpauth URI。

    参数说明：
      account   - 账号（邮箱或用户名）
      secret    - Base32 编码的密钥（从 app 获取）
      issuer    - 网站/服务名称（可选，但建议填写）
      algorithm - 哈希算法，绝大多数网站用 SHA1
      digits    - 动态码位数，绝大多数为 6
      period    - 刷新周期（秒），绝大多数为 30
    """
    secret = secret.strip().upper().replace(" ", "")

    # label 格式：issuer:account（有 issuer 时）或 account
    label = f"{issuer}:{account}" if issuer else account
    label = urllib.parse.quote(label, safe="")

    params = {
        "secret": secret,
        "algorithm": algorithm,
        "digits": str(digits),
        "period": str(period),
    }
    if issuer:
        params["issuer"] = issuer

    query = urllib.parse.urlencode(params)
    return f"otpauth://totp/{label}?{query}"


def generate_qr(uri: str, output_path: str = "totp_qrcode.png") -> None:
    """将 otpauth URI 渲染成二维码图片。"""
    qr = qrcode.QRCode(
        version=None,           # 自动选择最小版本
        error_correction=qrcode.constants.ERROR_CORRECT_M,  # 容错率 ~15%
        box_size=10,
        border=4,
    )
    qr.add_data(uri)
    qr.make(fit=True)

    img = qr.make_image(fill_color="black", back_color="white")
    img.save(output_path)
    print(f"✓ 二维码已保存：{output_path}")
    print(f"  URI：{uri}")


# ── 使用示例 ──────────────────────────────────────────────
if __name__ == "__main__":

    # 把从 app 获取的参数填进来
    ISSUER  = "GitHub"               # title / 网站名
    ACCOUNT = "user@example.com"     # 账号
    SECRET  = "JBSWY3DPEHPK3PXP"    # Base32 secret

    # 以下三项绝大多数网站都是默认值，除非 app 里有特别说明
    ALGORITHM = "SHA1"
    DIGITS    = 6
    PERIOD    = 30

    uri = build_otpauth_uri(
        account=ACCOUNT,
        secret=SECRET,
        issuer=ISSUER,
        algorithm=ALGORITHM,
        digits=DIGITS,
        period=PERIOD,
    )

    generate_qr(uri, output_path="totp_qrcode.png")
```
